## 「完成」の定義（前提）

- **「底に到達して倒す」型のクリアは設計と矛盾**（snapshot 4-3①：底は永久に未到達・到達不能を謎として温存）。
- よって「完成」＝(a) 確定システムが実装され、(b) 鋳造所コンテンツが十分量で組み合わせ空間が広大になり、(c) タイトル/導線/セーブ/バランス/配布が仕上がった、**オフライン買い切りPWA**（4-9E）。
- 動機づけの土台＝年代記（自分だけの歴史）の堆積。
- **方針：クリア／長期目標 → ★確定（snapshot 4-13「奉献の試練」）★（2026-06-15）。**

### クリア／長期目標 ＝「奉献の試練」（★確定 — snapshot 4-13）
- 「底は永久に未到達」（4-3①）を維持しつつ、**底に依存しないクリア/長期目標**を確定。クリア＝深度到達ではなく **①多様な偉業で資格を開く（gate）＋②帰還の試練（climax）** の二本柱。
- 先例の翻案：DCSS の Orb run（rune を集めて Zot 解禁→Orb 奪取→脱出）／NetHack のアセンション（最深で Amulet→地上まで帰還、追手あり）。本質は*帰還*で 4-3① と両立。当初案「深度到達だけで即クリア」は甘すぎるとして却下し、難しさの本体を「深度」→「多様な偉業」へ移した。
- 確定形態：
  - **(A) 5印（しるし）** … エリアボス撃破／因縁鎮魂／山場決着／旧キャラ伝説化／高深度到達。**World に蓄積し世代をまたぐ。** 揃って深淵帯が解錠。
  - **(B) 深淵帯＋聖遺物** … 解錠後の専用ダイブ。最奥の主が聖遺物を守る。
  - **(C) 帰還の試練** … 聖遺物携行で深蝕急騰＋追手。深度1→街へ生還でクリア。死ねば聖遺物は化石へ刻まれ後世が奪還（モデルC）。
  - **(D) 報酬／モデルC** … 達成（年代記）＋英雄譜＋聖遺物処遇（奉納＝街の恒久変化／佩用＝伝説級遺物）。印はクリアでリセットせず**反復可（H&S 継続）**、初期化時のみ0。複数回クリアで街が塗り替わる。
- 実装：**Phase1-4 完了**（印の収集→深淵帯/聖遺物→帰還の試練→メタ達成＝街の塗り替え）。Phase4＝PR #101（碑/像・貴族反応 arc `noble_ack`・統治者大命 `Quest.patron`／web限定・Lv45「原初の証」アークと別軸両立）。残＝数値バランス（テストプレイFB）。

### 冒険者イベントと同行 ＝「生者＝書きかけの化石」（★確定 — snapshot 4-14・2026-06-15）
- 生者の冒険者（`adventurers.md`・等級6段階 4-4E）を、ただの会話NPCでなく **モデルC（堆積する世界）の主役**にする。**(A) 背骨：出会う生者はいずれ化石になる。関わり方がその化石の"原稿"になり、化石・深蝕・世代交代・討つ/鎮めるの既存機構で後世に必ず再会する（亡霊・宿敵・伝説・聖遺物）。** 会話を「後世への書き込み」にして、ただ会話するNPCと差別化する＝本作の独自性の核。
- **(B) キャラ起点5類型：** A 同行／B 見殺し・救助（見捨て→怨念化→後世の grudge_hunt 宿敵を自分で執筆）／C 生者を討つ・鎮める（再会ごと深蝕進行→閾値で危険化→生者のうちに討つ(慈悲)/鎮める(心)）／D 深蝕の受け渡し（後段に温存）／E 約束と系譜の記憶（後段に温存）。**NPC依頼受注型は採らない**（ギルド経済 4-10G と二重・世界観的に不自然）。
- **(C) 同行＝相棒（Web版のみ・CLI/demo据え置き）：** 1体限定のグリッド相棒（@追従・隣接攻撃・テレグラフ・決定論）／連帯深蝕（潜行で相棒も深蝕↑→閾値で奇癖→C 発生）／死＝その床に絆つき化石ドロップ→後世で再会／救助で相棒化（B 融合）／生還で街に残存・等級↑・系譜記憶。再利用＝`meetActor`/`LivingActor`/`exposure`/`FossilEntity`＋再発見/`intervene`/`dungeon.ts`、**新規＝相棒エンティティ＋手番AI＋テレグラフ描画＋死亡時の化石ドロップ。**
- **実装の段階（コード側の次の一手）：**
  1. **context 場所分け（先に「器」を作る）** … `StoryletContext` を 6 種へ拡張＝`encounter`/`dungeon`/`street`/`tavern`/`guild`/`shop`（現状の `town` を `street` へ移行・`quest`/`chest` は別軸で併存可）。`selectStorylet`/`selectTownStorylet` に現在地パラメータを追加し呼び出し側で現在地を渡す。同一キャラを場所で別の顔に出せる。**先に器を作るとイベント投入が楽。M2。**
  2. **キャラ起点 B/C を 10〜15 人分** … 器が出来次第、メインシナリオ関連キャラから着手（横断A と合流）。
  3. **同行システム 4-14C** … 上記(C)。**✅ Phase A/B/C＋仕上げ(PR #109・2026-06-19) 実装完了（Web限定・`companion-check` 31 pass）。** 残り＝相棒固有 storylet の深掘り（任意）と数値バランス（テストプレイFB）。
  4. **D 深蝕の受け渡し／E 約束と系譜** … 後段（教団 P5・系譜深化と合流）。
- **コンテンツ生成の運用方針（3層分担・150人を一括で作らない／横断A に従う）：**
  - **種100** … 個別に作らない。**ランク×気質の共通テンプレ**で使い回す器。
  - **中核★50** … 「似合う型(A〜E)×登場シーン」だけ**関与度順**に作る（メイン優先）。
  - **ミスリル** … 年代記・英雄譜・化石テキストのみ。
  - storylet には **context を必ず付ける**。テンプレ変数（`#name#`/`#origin_name#`/`#origin_gear#` 等）を活用。
  - **実装済みの機構のテキストだけ作る**（未実装機構のテキストを先に量産しても使えない）。**当面はメインシナリオ関連キャラ10〜15人分のみ。** 残りは storylet/fossilScene/matchSetPiece/同行 の実装進捗に合わせて追加。
  - 依頼の仕方＝「キャラID＋似合う型＋場所(context)＋どの場面か」を指定し、その分だけ生成（一括生成はしない）。
