# design-spec.md — 見た目の仕様（方向A・移植の正）

「堆積する世界」のビジュアル仕様。**方向A＝発光グリフ・縦持ち・暗闇に滲む光**の言語化と、
散在していた色・字体・発光・コンポーネントの**意味づけしたトークン表**。

- **この文書が見た目の「正」。** web（`web/index.html` の `:root` トークン＋`src/web/main.ts` の描画）も
  将来の SwiftUI 版も、ここを実装する（4-9E＝製品は Swift。web は仕様＝リファレンス実装）。
- 値は現行実装から抽出した実値。**色を変えるときはここを更新してから**実装へ反映する。
- Swift 移植：このトークン表 → `Color` アセットカタログ／字体 → SF 系の対応フォント／発光 → `.shadow()` 多重がけ。

---

## 1. 方向A（ビジュアルの芯）

- **暗闇に発光するグリフ。** ほぼ黒（`--bg-void #07090c`）の盤面に、要素ごとの色で滲む光（多重 text-shadow）を置く。
- **色＝意味。** 色相は役割（自分=金／相棒=青／生者=緑／召喚・侵蝕=菫）と危険度（敵ティア 灰→黄→橙→赤→紫）を伝える。彩度・発光量が強いほど「重要・危険・特別」。
- **縦持ち単手。** 上＝情報（ステータス）、中＝盤面、下＝物語ログと親指操作（D-pad／タブバー）。
- **二つの声の字体。** 物語・ログ・シートは明朝（serif）で静かに、状態・システムはサンセリフで端的に、盤面はモノスペース。
- **動きは最小限の「呼吸」。** 重要物は緩い pulse、危険は速い danger、敵の予告は monatk で「来る」と知らせる（4-11A テレグラフ）。常時うるさく光らせない。

## 3. タイポグラフィ

3系統の字体で「声」を分ける。

| 役 | スタック | 用途 |
|---|---|---|
| sans | `"Hiragino Sans", system-ui, sans-serif` | 地色本文・ステータス・メタ |
| serif（明朝） | `"Hiragino Mincho ProN", "Yu Mincho", serif` | **物語・ログ・シート本文**（静かな語り） |
| mono | `"Menlo", "Consolas", monospace` | 盤面グリフ（等幅・字形の安定） |

サイズ（基準・px）：盤面 HUD 12／シート本文 14.5／ログ 15（設定で 13/15/17＝小中大）／ボタン 14.5／メタ 11／版数 10.5。
ウェイト：グリフと見出しは 700、本文は標準。行間：ログ 1.8・シート 1.95（読みやすさ優先）。

> Swift：serif→ヒラギノ明朝 or New York、mono→SF Mono/Menlo、sans→SF Pro/system。サイズは Dynamic Type に対応づけ。

## 4. 発光・動き（エレベーション）

- **発光＝二重 text-shadow。** 例：自分 `0 0 12px rgba(255,216,122,.9), 0 0 26px rgba(255,160,60,.45)`。内側で芯、外側で滲み。役割色ほど強く・広い。
- **アニメーション（`@keyframes`）**：
  - `pulse`（1.4–2.8s）＝重要物の「呼吸」（化石・泉・扉・召喚・t5）。
  - `danger`（.55s）＝被攻撃予告の自分/相棒が赤く脈打つ。
  - `monatk`（.5s）＝敵の攻撃予告（色はティアのまま強く明滅＝「来る」）。
  - `tele`（.55s）＝討たれるマスの赤枠／`tele-move`＝踏み込み先の琥珀ハイライト（4-11A）。
  - `fxflash`（.4–.55s）＝術の全域点滅（warp 菫／still 青／blink 青菫）。
- **光源ビネット** `#light`＝自分位置を中心にした暖色の radial（視界の芯を暖め、外周を闇に落とす）。
- 方針：**常時うるさく光らせない。** 静（床・壁は低彩度）と動（主役・危険）のコントラストで情報を立てる。

## 5.1 タイトル画面（横断F 段階1・確定＝変種A「灯火」）

起動の入口。**変種A 灯火**（自分＝金の光・暗闇に灯る暖色・「もう一度潜ろう」と迎える温度）を採用（ユーザー確定）。

- 構成（上→下）：火の粉（`embers`＝低不透明のモノ字グリフ散開）＋中心ビネット（暖色 radial）／ゲーム名「堆積する世界」＝大明朝＋金の発光（`#ffe6ad` / shadow `rgba(255,216,122,.55)`＋`rgba(255,160,60,.30)`）／タグライン「あなたの死が、世界の歴史になる。」（dim 明朝）／メニュー（下寄せ・最大幅340）／状態サブ行（続行中の主人公・深度／世代）／版数。
- メニュー：第1項目は **primary**（金グラデ `#ffdc8a→#e8b24a`・濃文字 `#1a1206`）＝「続きから」or「新しい物語」。他は半透明地＋微ブラー。
- 状態で出し分け：生存中＝続きから/新しい物語(=リセット)/設定。初回・死亡後＝新しい物語/設定。
- 実装：`web/index.html` `#title` オーバーレイ（z-index 30）＋`main.ts titleScreen()/titleChoose()`。本体は `townLoop→startDive` 連鎖へ一度分岐。
- Swift：`ZStack`（ビネット＋名＋メニュー）。金発光＝`.shadow()` 二層。primary はグラデ塗り。

## 5.2 HUD（ステータスバー・横断F 段階1・確定＝ゲージ化）

深蝕と HP を**文字列(░░░)から細いゲージへ**＝危険度が一目で読める（ユーザー確定で採用）。

- 2段＋バフ：①上段＝名前（主・太字・末尾省略）／`Lv` ／`深度`／（伸縮）／`◇`金貨／版数（右端固定＝最新判定）。②ゲージ段＝**HP**（朱グラデ・現/最大）と**深蝕**（菫グラデ・%。満タン=3.0＝旧5段×0.6基準）。③バフ段＝`⟡名 残手数`（潜行中のみ・空なら非表示）。
- ゲージ＝高さ6px・角丸・トラックは沈んだ地（`#0a0d12`＋`--line` インセット）、fill は左→右グラデ＋淡い発光。`width` を JS が設定（`transition .18s`）。
- 実装：`#status`（`#stTop`/`#stBars`/`#stBuff`）＋`updateStatus()`。
- Swift：`ProgressView` 相当を自前バー（角丸 + グラデ + shadow）で。色＝`--c-hp`/`--c-exp`。

## 6. Swift 移植ノート（横断F → M6）

- **色**：本トークン表 → `Assets.xcassets` の Color set（同名）。`:root` の hex をそのまま。発光の rgba は役割色から `opacity` 計算で再現。
- **発光**：CSS 二重 text-shadow → SwiftUI `.shadow(color:radius:)` の重ねがけ（内・外2層）。pulse 等は `withAnimation(.easeInOut.repeatForever())`。
- **字体**：serif/mono/sans を SF 系 or ヒラギノに対応づけ、サイズは Dynamic Type に。
- **レイアウト**：縦 `VStack`（status / board / log / deck / tabbar）、シートは `.sheet`/オーバーレイ。セーフエリアは `.ignoresSafeArea` ＋下端帯を `--bg-app` で塗る（iOS の既確定解＝CLAUDE.md 技術メモと同方針）。
- **盤面**：グリフは等幅フォントのテキスト or 軽量描画。色＝役割トークン、発光＝shadow。

> 視覚の「方向」は本書が正。実装は web（リファレンス）→ Swift（製品）の順でミラーする。数値の最終調整（横断F）は実機で。
